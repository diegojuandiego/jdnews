El trabajo para Web 1 consistía en un sitio web símil portal de noticias.
En este caso, apuntaba como su nombre lo indica a ciencias y tecnología.

El mismo consta de las siguientes páginas:

  • Inicio/Noticias: El feed completo de noticias del sitio.
  • Destacadas: Una selección de noticias destacadas bajo un criterio no informado.
  • Sobre el proyecto: Esta sección. Una breve reseña del diagramado del sitio.
  • Contacto: Un apartado con algunos medios de contacto.

El feed de noticias

El listado de noticias fue extraído del feed RSS de Tecnología de la agencia Télam. Curiosamente las noticias son viejísimas por lo que deduzco que dicho feed está discontinuado.

https://www.telam.com.ar/rss

El display de noticias

El componente "news list"" es un bucle ngFor de una plantilla tipo "tarjeta" con información sobre las noticias.

El typescript carga el feed de noticias en una variable que será iterada, llamando al servicio "news serve" que devuelve un arreglo, imitando a una API.

Al hacer click en una noticia, es capaz de cargarla en el servicio "news store" (a través de un método del mismo) que utilizamos a modo de almacenamiento.

Noticias destacadas

Algunas noticias están marcadas o destacadas mediante el atributo homónimo. En el sitio se pueden ver con el tag DESTACADA.

El servicio que usamos como reservorio para la noticia a mostrar, "news store", también almacena una propiedad que determina si el contenido debe ser filtrado. Esta última se modifica mediante métodos que expone dicho servicio cada vez que elegimos mostrar la lista completa de noticias, o sólo las destacadas, mediante un evento "(click)" en la barra de navegación.

Título... ¿Desapercibido?

En Angular, el título de la página queda determinado a la hora de creación del proyecto, a tal punto que ni siquiera se puede modificar en el atributo "title" de app.component.html. Para ello se inyectó en cada componente que lo requiera el servicio que provee Angular para sortear esta situación. Con un método que provee dicho servicio se modifica el título del sitio para hacer la experiencia más amena y considerada para con el usuario.

Ruteo

Se utilizó el servicio de Ruteo de Angular para evitar que el usuario pueda acceder a rutas permitidas por la aplicación pero que sin la lógica no funcionarían.

En este caso particular, la ruta "/read" (visualización de una noticia) permite visualizar la selección si la lógica la maneja la aplicación. Pero si el usuario la utiliza, la pantalla quedaría en blanco. Para evitar ese bug se redirecciona la aplicación a la página "home".
